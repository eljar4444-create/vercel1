# Plan 01: Extend Design Tokens & Base CSS

<objective>
Extend Tailwind config and global CSS with all design tokens required by the WIREFRAME.md:
glassmorphism variables, the Jealousy Card shimmer keyframe, and the manifesto entrance animation.
This wave has zero UI changes — it only extends the design system foundation.
</objective>

<context>
@_bmad-output/brainstorming/WIREFRAME.md (Section 1: Global Design Rules)
@tailwind.config.ts
@src/app/globals.css
</context>

<tasks>

<task>
<description>Add glassmorphism color tokens to Tailwind config</description>
<read_first>
- tailwind.config.ts
</read_first>
<action>
In `tailwind.config.ts` under `theme.extend.colors.booking`, add:

```typescript
glass: "rgba(255, 255, 255, 0.40)",
glassBorder: "rgba(255, 255, 255, 0.60)",
```

These tokens are used by the Jealousy Card (Act 4) and any future glassmorphism elements.
</action>
<acceptance_criteria>
- `tailwind.config.ts` contains `glass: "rgba(255, 255, 255, 0.40)"`
- `tailwind.config.ts` contains `glassBorder: "rgba(255, 255, 255, 0.60)"`
- `npx tailwindcss --help` exits 0 (config is valid)
</acceptance_criteria>
</task>

<task>
<description>Add shimmer keyframe and manifesto entrance animation to Tailwind config</description>
<read_first>
- tailwind.config.ts
</read_first>
<action>
In `tailwind.config.ts` under `theme.extend.keyframes`, add:

```typescript
"glass-shimmer": {
  "0%": { backgroundPosition: "-200% 0" },
  "100%": { backgroundPosition: "200% 0" },
},
"manifesto-enter": {
  "0%": { opacity: "0", transform: "translateY(20px)" },
  "100%": { opacity: "1", transform: "translateY(0)" },
},
```

Under `theme.extend.animation`, add:

```typescript
"glass-shimmer": "glass-shimmer 2s ease-in-out 1",
"manifesto-enter": "manifesto-enter 0.6s ease-out forwards",
```
</action>
<acceptance_criteria>
- `tailwind.config.ts` contains `"glass-shimmer"` in both `keyframes` and `animation`
- `tailwind.config.ts` contains `"manifesto-enter"` in both `keyframes` and `animation`
</acceptance_criteria>
</task>

<task>
<description>Add utility CSS classes for neumorphic button press and glassmorphism shimmer overlay</description>
<read_first>
- src/app/globals.css
</read_first>
<action>
At the end of `src/app/globals.css` (inside `@layer utilities` or after existing rules), add:

```css
@layer utilities {
  .btn-neu {
    @apply shadow-soft-out transition-all duration-100;
  }
  .btn-neu:active, .btn-neu.pressed {
    @apply shadow-soft-in scale-[0.98];
  }

  .glass-card {
    @apply bg-white/40 backdrop-blur-[12px] border border-white/60 shadow-glass rounded-[2rem];
  }

  .glass-shimmer-overlay {
    background: linear-gradient(
      90deg,
      transparent 0%,
      rgba(255, 255, 255, 0.15) 50%,
      transparent 100%
    );
    background-size: 200% 100%;
  }
}
```
</action>
<acceptance_criteria>
- `src/app/globals.css` contains `.btn-neu`
- `src/app/globals.css` contains `.glass-card`
- `src/app/globals.css` contains `.glass-shimmer-overlay`
- `npm run build` completes without CSS errors
</acceptance_criteria>
</task>
